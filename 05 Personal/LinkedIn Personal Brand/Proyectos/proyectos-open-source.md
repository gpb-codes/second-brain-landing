# Proyectos Open Source

## Filosofía Open Source

Creo que el conocimiento técnico debe ser accesible. Por eso, siempre que es posible, publico proyectos con licencias open source. No solo código: documentación, plantillas, guías y frameworks.

## Cómo contribuyo al open source

1. **Código** — PRs con features, fixes y mejoras
2. **Documentación** — README, guías, ejemplos
3. **Issues** — Reportes detallados de bugs
4. **Discusiones** — Participación en comunidades técnicas
5. **Traducciones** — Documentación en español

---

## Por qué open source

| Razón | Explicación |
|-------|-------------|
| Aprendizaje | El mejor código es el que otros leen |
| Networking | Las mejores conexiones vienen de colaborar |
| Portafolio | Tu GitHub habla por ti |
| Impacto | Tu código puede ayudar a alguien al otro lado del mundo |
| Calidad | El open source te obliga a escribir mejor código |

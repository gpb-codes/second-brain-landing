# Entrevistas — Preparacion para cada plataforma

## LinkedIn — Perfil como pre-entrevista

Tu perfil de LinkedIn es la primera impresion antes de cualquier entrevista.

### Checklist pre-entrevista
- [ ] Perfil actualizado (experiencia, skills, projects)
- [ ] Foto profesional reciente
- [ ] Banner personalizado
- [ ] Recomendaciones de colegas
- [ ] Publicaciones recientes visibles
- [ ] Proyectos destacados actualizados

### Que los reclutadores revisan en tu perfil
1. Titular: la primera linea de texto
2. Acerca de: el storytelling profesional
3. Proyectos: demuestran iniciativa y habilidad practica
4. Skills: aunque seas estudiante, las skills validadas importan
5. Actividad: publicaciones e interacciones recientes
6. Formacion: la carrera en curso mas proyectos = perfil atractivo

---

## Instagram — No aplica directamente

Instagram no es una plataforma de busqueda laboral directa, PERO los reclutadores pueden revisarlo.

### Que cuidar en Instagram
- [ ] Bio profesional sin controversial
- [ ] Contenido tecnico que refuerce tu perfil
- [ ] Sin publicaciones que resto profesionalismo
- [ ] Highlights curados con tu trabajo
- [ ] Historia destacada "Sobre mi"

### Como usarlo a favor
- Publica contenido tecnico que demuestre expertise
- Muestra detras de escena de proyectos
- Conecta con reclutadores via DMs

## GitHub — La entrevista tecnica silenciosa

Tu GitHub habla antes que tu en una entrevista tecnica.

### Que los reclutadores tecnicos buscan en GitHub
| Elemento | Que evalua | Que deben ver |
|----------|------------|---------------|
| Perfil | Seriedad | Foto, bio, pinned repos |
| Commits | Consistencia | Actividad regular |
| README | Comunicacion | Claridad y documentacion |
| Codigo | Calidad | Clean code, buenas practicas |
| PRs | Colaboracion | Contribuciones externas |
| Issues | Resolucion | Problemas bien reportados |

### Estrategia pre-entrevista
1. **Revisa tu GitHub como si fueras reclutador**
2. **Asegura commits recientes** (actividad en las ultimas 2 semanas)
3. **README de proyectos impecables** (documentacion clara)
4. **Code style consistente** (usa linters, formatters)
5. **Elimina repositorios incompletos** o ponles "archived"

### Lenguajes y tecnologias a mostrar
Segun el puesto:
- IA/ML: Python, Jupyter, TensorFlow/PyTorch, LangChain
- Automatizacion: Python, Bash, GitHub Actions, n8n
- Desarrollo Web: TypeScript, React, Node.js, APIs

### Preguntas tecnicas comunes + respuesta desde proyectos

| Pregunta | Como responder con tus proyectos |
|----------|----------------------------------|
| "Hablanos de un proyecto complejo" | Drakkar Labs: arquitectura, decisiones, resultados |
| "Como manejas errores" | Pipeline de automatizacion: logging, fallbacks |
| "Experiencia con APIs" | Integracion de OpenAI, Claude, Notion, GitHub |
| "Trabajo en equipo" | Contribuciones open source, PRs |
| "Metodologia" | Proyectos personales con CI/CD, git flow |

### Preguntas de comportamiento (STAR)

**"Cuentame de un proyecto que lideraste"**
```
S: Queria construir un asistente IA pero no tenia equipo
T: Necesitaba disenar, desarrollar y lanzar solo
A: Use Drakkar Labs como framework, priorice features clave, decidi stack
R: Prototipo funcional en 1 fin de semana, 94% reduccion de tiempo en tareas
```

**"Como aprendiste IA?"**
```
S: No tenia formacion formal en IA
T: Necesitaba dominar prompt engineering e integracion de LLMs
A: Cree proyectos practicos: asistente Obsidian, pipeline automatizacion
R: Hoy integro APIs de OpenAI, Claude, y modelos locales en produccion
```

**"Fracaso y aprendizaje"**
```
S: Primer proyecto personal: quise hacerlo perfecto
T: Pase 3 meses puliendo features que nadie pidio
A: Aprendi a lanzar rapido, iterar con feedback
R: Ahora lanzo prototipos en <1 semana, mejoro basado en uso real
```

## Preguntas frecuentes por rol

### Para puesto de IA/ML
1. "Explica la diferencia entre IA, ML y Deep Learning"
2. "Como manejas el prompt engineering?"
3. "Has trabajado con RAG?"
4. "Modelos locales vs cloud: cuando usar cual?"
5. "Como evaluas la calidad de un output de IA?"

### Para puesto de automatizacion
1. "Que has automatizado y como?"
2. "Como decides QUE automatizar?"
3. "Herramientas de automatizacion que domines"
4. "Errores comunes en automatizacion"
5. "Como aseguras que la automatizacion sea confiable?"

### Para puesto de desarrollo web
1. "Arquitectura que usas en tus proyectos"
2. "Backend: REST vs GraphQL?"
3. "Testing: que y como pruebas?"
4. "Bases de datos: SQL vs NoSQL?"
5. "Deployment: tu pipeline ideal"

## Preguntas especificas para estudiantes/practicantes

1. "Por que decidiste estudiar Ingenieria Informatica?"
2. "Que proyectos has construido por tu cuenta?"
3. "Como compaginas estudio con proyectos personales?"
4. "Que tecnologia te apasiona y por que?"
5. "Donde te ves en 2-3 anos?"
6. "Que has aprendido por tu cuenta que no te hayan ensenado en la carrera?"
7. "Como te mantienes actualizado en tecnologia?"
8. "Has trabajado en equipo en algun proyecto?"
9. "Que parte de la ingenieria de software te gusta mas?"
10. "Por que deberiamos contratarte como practicante?"

### Como responder siendo estudiante

| Situacion | Enfoque de respuesta |
|-----------|---------------------|
| Poca experiencia laboral | Habla de proyectos personales, Drakkar Labs, open source |
| Conocimiento teorico limitado | Enfatiza aprendizaje autonomo y curiosidad |
| "Apenas vas en primer semestre" | Muestra iniciativa: proyectos > semestres cursados |
| Falta de titulo | Demuestra con codigo, proyectos y resultados |

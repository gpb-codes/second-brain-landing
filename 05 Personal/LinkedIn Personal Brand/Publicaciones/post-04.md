# Post 04 — Mi stack de productividad técnica (2026)

**Formato:** Lista / Recomendación
**Tono:** Práctico, directo
**Pilar:** Productividad técnica

---

Después de probar decenas de herramientas, este es mi stack actual:

—

### 🧠 Segundo Cerebro — Obsidian
Mi base de conocimiento principal.
- 1,200+ notas interconectadas
- Sistema de tags + MOCs (Maps of Content)
- Plugins clave: Dataview, Kanban, Excalidraw
- Sincronizado con Git

### ✍️ Escritura técnica — VS Code + Obsidian
- Escribo borradores en Obsidian
- Edito y publico desde VS Code
- Markdown en todos lados

### 🤖 IA — Claude + GPT + Local Models
- Claude para escritura y análisis
- GPT para tareas diversas
- Modelos locales (Ollama) para experimentación

### 📂 Proyectos — GitHub + Notion
- GitHub para código y documentación técnica
- Notion para roadmap y seguimiento

### ⚡ Automatización — Python + n8n
- Scripts en Python para tareas repetitivas
- n8n para workflows visuales
- APIs personalizadas

### 📱 Comunicación — LinkedIn + Twitter/X
- LinkedIn para contenido profesional
- Twitter para micro-contenido técnico
- Newsletter (próximamente)

—

### Lo más importante

No es la herramienta, es el **sistema**.

Puedes tener el mejor stack del mundo.
Si no tienes un flujo de trabajo consistente, no sirve de nada.

—

Mi regla de oro: **cada herramienta resuelve un problema específico**. Si no sé qué problema resuelve, no la uso.

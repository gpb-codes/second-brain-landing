# Plantillas para Repositorios de GitHub

## Plantilla 1: README de proyecto

```markdown
# [Nombre del Proyecto]

> [Descripción corta y potente — 1 línea]

## 🚀 Descripción

[2-3 párrafos explicando qué hace el proyecto, por qué existe y qué problema resuelve.]

## ✨ Características

- [Característica 1]
- [Característica 2]
- [Característica 3]
- [Característica 4]

## 🛠️ Stack

| Tecnología | Versión | Uso |
|------------|---------|-----|
| Python | 3.12+ | Core |
| [Framework] | [versión] | [uso] |

## 📦 Instalación

```bash
git clone https://github.com/gpedreros/[repo].git
cd [repo]
pip install -r requirements.txt
```

## ⚡ Uso rápido

```python
from project import main
main()
```

## 📁 Estructura

```
├── src/
│   ├── main.py
│   └── utils.py
├── tests/
├── docs/
├── requirements.txt
└── README.md
```

## 🤝 Contribuciones

Las contribuciones son bienvenidas. Por favor:

1. Fork el proyecto
2. Crea tu rama (`git checkout -b feature/AmazingFeature`)
3. Commit (`git commit -m 'Add AmazingFeature'`)
4. Push (`git push origin feature/AmazingFeature`)
5. Abre un Pull Request

## 📄 Licencia

Distribuido bajo [Licencia]. Ver `LICENSE` para más información.

## 📫 Contacto

Gabriel Pedreros — [@gpedreros](https://github.com/gpedreros)

---

⭐️ Si este proyecto te fue útil, ¡dale una estrella!
```

## Plantilla 2: CONTRIBUTING.md

```markdown
# Guía de Contribución

## Cómo contribuir

1. 🍴 Fork del repositorio
2. 🌿 Crea tu rama de feature
3. 💻 Escribe código siguiendo el estilo del proyecto
4. ✅ Asegúrate de que los tests pasen
5. 📝 Describe bien tu Pull Request

## Estándares de código

- Sigue las convenciones del lenguaje
- Comenta solo cuando sea necesario
- Escribe tests para nuevas funcionalidades
- Usa nombres descriptivos

## Pull Requests

- Describe qué cambia y por qué
- Referencia issues relacionados
- Mantén PRs pequeños y enfocados
- Responde a los comentarios de revisión
```

## Plantilla 3: ISSUE_TEMPLATE.md

```markdown
---
name: Reporte de bug
about: Crea un reporte para ayudarnos a mejorar
title: ''
labels: bug
assignees: ''
---

**Descripción del bug**
Una descripción clara y concisa del error.

**Para reproducir**
Pasos:
1. Ir a '...'
2. Hacer clic en '...'
3. Error: '...'

**Comportamiento esperado**
Lo que debería ocurrir.

**Capturas**
Si aplica, añade capturas de pantalla.

**Entorno:**
- OS: [ej: Windows 11]
- Versión: [ej: 1.0.0]

**Contexto adicional**
Cualquier otro detalle relevante.
```

- [ ] Manualmente

## Checklist

- [ ] Se han añadido tests
- [ ] La documentación está actualizada
- [ ] No hay warnings nuevos

## Issues relacionados

Closes #[issue]
```

# Guía Open Source para GitHub

## Estrategia de contribuciones

### Fase 1: Fundamentos (Mes 1-2)
- Crear repositorios personales con buena documentación
- READMEs completos, licencias, .gitignores
- Escribir código limpio y comentado
- Consistencia diaria de commits

### Fase 2: Contribuciones externas (Mes 3-4)
- Encontrar proyectos afines (IA, automatización, productividad)
- Empezar con issues etiquetados como "good first issue"
- Hacer PRs pequeños y bien documentados
- Participar en discussions

### Fase 3: Comunidad (Mes 5-6)
- Los proyectos propios empiezan a recibir contribuciones
- Responder issues y PRs de otros
- Compartir en redes los proyectos
- Colaboraciones con otros devs

### Filosofía como mantenedor

1. **Respuesta rápida** — Issues y PRs en <48h
2. **Código de conducta** — Comunidad respetuosa
3. **Documentación clara** — Si no está documentado, no existe
4. **Revisión constructiva** — Feedback que ayuda a crecer
5. **Reconocimiento** — Contributors en README

## Buenas prácticas

### Lo que SÍ hacer
✅ README completo con badges
✅ Licencia clara (MIT preferida)
✅ Issues con templates
✅ GitHub Actions para CI/CD
✅ Responder a toda contribución
✅ Celebrar a los contributors

### Lo que NO hacer
❌ README vacío o genérico
❌ Sin licencia
❌ Ignorar issues o PRs
❌ Código sin tests
❌ No tener .gitignore
❌ Branches mal nombrados

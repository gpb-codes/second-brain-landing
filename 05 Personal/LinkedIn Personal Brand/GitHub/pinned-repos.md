## Cómo escribir descripciones efectivas

| Elemento | Recomendación |
|----------|---------------|
| **Primera línea** | Qué hace el proyecto (máx 100 chars) |
| **Tono** | Activo: "Automatiza X", "Crea Y", "Organiza Z" |
| **Keywords** | Términos que la gente buscaría |
| **Emoji** | 1-2 como máximo |
| **Links** | Website o demo si aplica |

### Ejemplos buenos vs malos

| ❌ Mal | ✅ Bien |
|--------|---------|
| "Un proyecto de IA" | "Automatiza tareas repetitivas con IA usando Python y OpenAI" |
| "Herramienta genérica" | "Organiza tus notas en Obsidian con este template de PKM" |
| "Cosas que hice" | "Framework de prompts para ingenieros que usan LLMs" |

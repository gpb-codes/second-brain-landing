# Hooks para Publicaciones en LinkedIn

## Qué es un hook

Las primeras 3 líneas de tu publicación (lo visible sin hacer clic en "ver más"). Determinan si alguien para o sigue scrolleando.

---

## Fórmulas de hooks probadas

### 1. La pregunta provocadora
```
¿Cuál es el verdadero riesgo de la IA?
No es que los modelos sean más inteligentes que nosotros.
Es que no aprendas a usarlos.
```

### 2. La cifra impactante
```
4 horas semanales perdidas en documentación.
15 minutos después de automatizar con IA.
El mismo resultado, 94% menos tiempo.
```

### 3. El error común
```
El error más común al aprender a programar no es técnico.
No es elegir el lenguaje equivocado.
Es caer en el tutorial hell.
```

### 4. La confesión honesta
```
Construí mi primer proyecto durante 3 meses.
Lo usaron 4 personas (yo incluido).
Fue el mejor aprendizaje de mi carrera.
```

### 5. El "nadie habla de esto"
```
Nadie te dice que emprender en tech es 10% código y 90%...
Gestión emocional, paciencia y consistencia.
Esto es lo que nadie te cuenta.
```

### 6. La comparativa directa
```
Antes: 4 horas documentando reuniones.
Después: 15 minutos con IA.
El cambio no fue la herramienta, fue el proceso.
```

### 7. La afirmación controversial
```
La IA no va a reemplazar a los ingenieros.
Los ingenieros que usan IA van a reemplazar a los que no.
¿Suena duro? La historia dice lo mismo.
```

### 8. El "cómo" prometedor
```
Cómo construí un asistente IA en un fin de semana.
Sin equipo, sin presupuesto, sin experiencia previa.
Solo código, APIs y ganas de experimentar.
```

### 9. La historia personal
```
Hace un año estaba atascado en mi zona de confort técnica.
Hoy tengo proyectos, comunidad y dirección clara.
Este fue el cambio que lo inició todo.
```

### 10. El dato curioso
```
Sabías que el 80% del tiempo en reuniones técnicas se pierde en...
Transcripción, documentación y seguimiento.
Automaticé eso y recuperé mi semana.
```

## Banco de hooks (50+)

### Sobre IA
- "La mayoría usa IA como una calculadora. Los ingenieros la usan como un socio."
- "Usar ChatGPT no es saber de IA. Saber cuándo NO usarla, tampoco."
- "El prompt más importante que escribirás hoy no es para la IA. Es para ti mismo."
- "3 años de evolución IA en 3 meses: así se siente 2026."
- "No necesitas más herramientas de IA. Necesitas mejores procesos."

### Sobre desarrollo
- "Tu código se lee más veces de las que se escribe. Escríbelo para humanos."
- "El mejor consejo técnico que recibí: escribe código que puedas explicar en 5 segundos."
- "No hay código perfecto. Hay código que funciona y se mantiene."
- "La deuda técnica es como la deuda financiera: los intereses se acumulan."

### Sobre productividad
- "No es falta de tiempo. Es falta de sistema."
- "Tener 10 herramientas de productividad es lo opuesto a ser productivo."
- "El mejor software de productividad no existe: el que construyes para ti mismo."
- "Tu segundo cerebro no sirve si no confías en él."

### Sobre emprendimiento
- "El momento perfecto para empezar no existe. Llevo años esperándolo."
- "Emprender en tech no es sexy. Es incómodo, incierto y adictivo."
- "Tu primer proyecto no necesita ser original. Necesita ser útil."
- "El fracaso en proyectos no es fracaso. Es data."

### Sobre aprendizaje
- "Aprender a programar es fácil. Aprender a terminar proyectos es lo difícil."
- "El tutorial hell es real. La cura es un proyecto que te importe."
- "No necesitas otro curso. Necesitas un problema que resolver."
- "Enseñar es el mejor atajo para aprender."

## Hook por formato de publicación

| Formato | Tipo de hook recomendado |
|---------|------------------------|
| Opinión | Provocador / Controversial |
| Tutorial | Promesa de valor / "Cómo" |
| Historia | Confesión / Personal |
| Lista | Cifra / "Nadie habla de esto" |
| Caso de estudio | Comparativa Antes/Después |
| Reflexión | Pregunta / Afirmación |
